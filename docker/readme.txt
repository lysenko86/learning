docker - дуже низькорівневий, треба пам'ятати купу атрибутів
docker-compose - більш високорівневий, він менеджить docker, тут описуємо інструкції як і що docker має робити
    - треба щоб оголошувати і запускати контейнери докера

Мікросервісна архітектура
    - ізольовані частині коду, з тільки своїми потрібними залежностями
    - можуть бути написані на різних мовах
    - можуть розроблятись окремо і незалежно
    - до прикладу може бути так:
          - Сервіс 1 - Фронтенд (React)
          - Сервіс 2 - API (Node)
          - Сервіс 3 - API авторизації (Node)
          - Сервіс 4 - API БД (Mongo)
          - Сервіс 5 - API авторизації БД (Mongo)
          - Сервіс 6 - Nginx (proxy)

docker-compose.yml - файл-конфігурації docker-compose

Dockerfile - лежить в кожному сервісі і описує інструкцію для білда образа цього сервісу

docker image - це образ лінукса, до якого додаємо все що нам потрібно в контейнері

=== Команди
    - docker images - виводить всі images на нашій локальній машині
    - docker-compose build - білдить образ згідно з файлом Dockerfile
    - docker-compose up - запуск збілдженого контейнера
