Kubernetes - оркестрація контейнерів в кластері

Структура кластеру (worker nodes, або міньйони а також мастер-нода):
    - Master-node
        - API server - фронтенд, інтерфейс для Kubernetes
        - ETCD - storage ключ-значення, розподілена база даних, тут зберігається вся інформація по керуванню кластером
        - Scheduler - розподіл робіт, або контейнерів між нодами, чекає коли з'явиться новий контейнер в системі, щоб назначити його виконання на ноду
        - Controller - мозок оркестрації - стан nodes, контейнерів, ендпоінтів, реагують на події на нодах, також приймають рішення про створення нових контейнерів
    - Worker-node 1 - сервер для розміщення контейнерів
        - Kubelet - агент, який працює на кожному вузлі кластера, відповідає за те, щоб контейнери справно працювали на вузлах
            - співпрацює з API server масетр-ноди
        - Container Runtime - середовище виконання контейнерів, базове ПО для запуска контейнерів, наприклад Docker
    - Worker-node 2
        - Kubelet
        - Container Runtime
    - Worker-node 3
        - Kubelet
        - Container Runtime

Утиліта kubectl - юзається для розгортання і керування кластерами
    - kubectl run hello-world - розгортає аплікуху hello-world в кластері
    - kubectl cluster-info - інформація про кластер
    - kubectl get nodes - вивід всіх вузлів (нод) в кластері

Minikube
    - утиліта, щоб розібратись, спробувати Kubernetes
    - створює одновузловий кластер Kubernetes
    - всі частини в одному місці: API server, ETCD, Scheduler, Controller, Kubelet, Container Runtime

Kubeadm
    - дозволяє побудувати і налаштувати кластери рівня prod
